# Atelier — Virtual Fitting Room (نسخة PoC بـ Backend حقيقي)

موقع Multi-page شغال بالكامل: صفحة رئيسية، استمارة قياس، نتيجة (مقاس + أفاتار +
ألوان + تأكيد طلب)، وصفحة تتبع طلب — مبني على Node.js بدون أي مكتبات خارجية،
وبيحفظ الطلبات فعليًا في قاعدة بيانات SQLite على القرص (مش في المتصفح).

## المتطلبات
- Node.js نسخة 22 أو أحدث (بيستخدم `node:sqlite` المدمجة في Node، مفيش أي
  `npm install` مطلوب خالص).

## التشغيل محليًا
```bash
cd atelier-virtual-fitting-room
node server.js
```
هيظهر لك:
```
✅ الموقع شغال على http://localhost:3000
```
افتح الرابط ده في المتصفح.

## البنية
```
server.js         السيرفر (HTTP + كل الـ API routes)
sizeEngine.js      محرك حساب المقاس (المصدر الوحيد للحساب، مش بيتكرر في المتصفح)
db.js              طبقة قاعدة البيانات (SQLite) — إنشاء/قراءة/تحديث الطلبات
db/orders.db       ملف قاعدة البيانات نفسه (بيتعمل تلقائيًا أول تشغيل)
public/            كل صفحات وملفات الموقع (HTML/CSS/JS يشوفها المتصفح)
```

## الـ API
| Method | المسار                          | الوظيفة                                   |
|--------|----------------------------------|--------------------------------------------|
| POST   | `/api/calculate`                 | يحسب المقاس وينشئ طلب جديد (draft)          |
| GET    | `/api/orders/:code`               | يجيب بيانات طلب بكود التتبع                 |
| PATCH  | `/api/orders/:code/colors`        | يحدّث ألوان الجاكيت/القميص/البنطلون/الحذاء   |
| POST   | `/api/orders/:code/confirm`       | يأكّد الطلب (بيطلب اسم وتليفون)              |

## ملاحظات مهمة قبل أي إطلاق فعلي
1. **جدول المقاسات** (`SIZE_CHART` في `sizeEngine.js`) دلوقتي مبني على جدول
   قياسي عالمي تقريبي (EU 44-60). لو عندك جدول مقاسات حقيقي من البراند اللي
   هتشتغل معاه، استبدل الأرقام دي بأرقامه مباشرة — باقي الكود مش هيحتاج تغيير.
2. **قاعدة البيانات** SQLite ملف واحد (`db/orders.db`) — ده تمام للـ PoC
   وللتجربة المحلية، لكن لو هتنشر أونلاين بعدد مستخدمين كبير، الخطوة الطبيعية
   بعد كده الانتقال لقاعدة بيانات سيرفر (Postgres/MySQL) — البنية الحالية
   (`db.js`) مفصولة عن باقي الكود عشان الاستبدال ده يبقى سهل.
3. **لسه من غير لوحة تحكم أدمن** — اتفقنا عليها كخطوة تالية منفصلة.
4. **النشر أونلاين**: لسه محليًا بس دلوقتي زي ما اتفقنا. لما تحب تنشره فعليًا،
   محتاج استضافة بتدعم Node.js (زي Render أو Railway أو VPS عادي) + دومين.
